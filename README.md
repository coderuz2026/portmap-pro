# PortMap Pro — как собрать APK для Google Play

## Что умеет
- Документация портов свитча: статус, устройство, VLAN, кабель ID, локация, заметка
- Реалистичная панель портов, поиск/фильтр по сети
- Регистрация и вход (локально, офлайн)
- **QR на порт** + **сканер QR камерой** → мгновенный прыжок к порту
- **PDF-отчёт** по свитчу и **лист QR-наклеек** на печать
- **CSV (Excel)** импорт-экспорт + JSON бэкап
- **Патч-панели** как отдельный тип объекта + **связь портов** (свитч ↔ панель ↔ розетка) и **трасса кабеля**
- **📸 Фото-разметка** — метки портов поверх реального снимка свитча (тап по метке → карточка порта)
- **⚡ Impact-анализ** — «что отвалится» вниз по трассе при отключении порта/объекта
- **🗺️ Авто-топология** — схема сети строится сама из связей портов
- **📶 NFC-метки** — запись/чтение метки на кабель (Android/Chrome, работает в TWA)
- **🤖 AI-диагностика** — анализирует ТВОЮ топологию: что упало, почему, что отвалится
- **🌗 Смена темы** — тёмная / светлая
- **Облако (Firebase)** — опционально, для аккаунтов и синка между телефонами

## AI в боевом APK
В превью Claude.ai AI работает сразу. Для опубликованного приложения:
- НЕ вшивай API-ключ в код (его увидят все, кто распакует APK)
- Подними лёгкий прокси (Cloudflare Worker / Vercel), который хранит ключ и дергает api.anthropic.com
- Впиши адрес прокси в `AI.proxyUrl` в `index.html`
- Либо, для личного использования — вставь свой ключ в `AI.apiKey` (только если не раздаёшь APK)
- Free / Pro гейт (Pro = безлимит объектов)

## Включить облако (по желанию)
Открой `index.html`, найди блок `const CLOUD`. По шагам:
1. console.firebase.google.com → создай проект
2. Authentication → Email/Password → включить
3. Firestore Database → создать
4. Вставь свой `firebaseConfig` и поставь `enabled:true`
По умолчанию облако выключено — приложение полностью работает офлайн.

Приложение готово как PWA. APK собирается из него за ~3 минуты, без Android Studio.

## Шаг 1. Залить на хостинг (HTTPS обязателен)
У тебя уже есть GitHub Pages. Положи всю папку в репозиторий, например:

    coderuz2026.github.io/portmap-pro/

Проверь, что открывается:
- https://coderuz2026.github.io/portmap-pro/index.html
- https://coderuz2026.github.io/portmap-pro/manifest.json

## Шаг 2. Собрать APK через PWABuilder
1. Открой https://www.pwabuilder.com
2. Вставь URL своего PWA → **Start**
3. Дождись отчёта (manifest + service worker должны быть «зелёными»)
4. **Package For Stores → Android → Generate**
5. Скачаешь zip с двумя файлами:
   - `app-release-signed.apk` — поставить на телефон для теста
   - `app-release-bundle.aab` — загрузить в Google Play Console
   - `assetlinks.json` — **обязательно** положить по пути:
     `coderuz2026.github.io/.well-known/assetlinks.json`
     (иначе в TWA вверху будет видна адресная строка браузера)

## Шаг 3. Подпись (важно — не потеряй ключ!)
PWABuilder генерит keystore. Сохрани:
- `signing.keystore`
- пароли и алиас из `signing-key-info.txt`

Эти же ключи нужны для КАЖДОГО будущего обновления. Потеряешь — не сможешь обновлять приложение в Play под тем же листингом.

## Шаг 4. Google Play Console
- Аккаунт разработчика: разовая оплата $25
- Создать приложение → загрузить `.aab`
- Заполнить: описание, скриншоты (мин. 2), иконка 512×512 (есть в `icons/`),
  feature graphic 1024×500, политика конфиденциальности (нужна ссылка)
- Возрастной рейтинг, страна, цена (free + Pro через Play Billing)

## Аккаунты / вход
Сейчас вход локальный (имя + пароль, хранится на устройстве, офлайн).
Для облачных аккаунтов с входом на разных телефонах — следующий шаг через Firebase Auth.

## Полировка перед релизом (по желанию)
- Вынести шрифты и qrcode.min.js локально (сейчас грузятся с CDN; кешируются service worker'ом, но для 100% офлайна лучше положить рядом)
- Подключить реальный Google Play Billing вместо демо-кнопки Pro
- Прикрутить SNMP-синк и GitHub-сохранение из старого PortMap

## Обновление (v2)
- **Транк-порты** — режим Access/Trunk на порту свитча, разрешённые VLAN, метка «T» на панели портов
- **Импорт вывода команд** — вставь `show mac address-table` / `show interface status` / `show cdp neighbors` → порты заполнятся сами (меню объекта)
- **Массовое редактирование** — задать статус/режим/VLAN сразу диапазону портов
- **Шаблоны объектов** — BDCOM/Cisco/патч-панель в один тап при создании
- **Иерархия площадок** — поля «Площадка» и «Шкаф», группировка на главном экране
- **QR на объект** — наклейка на дверцу шкафа, скан открывает весь объект
- **Демо-сеть** — в меню, для быстрого просмотра AI/impact/топологии
- **Сканер**: добавлен режим «Из фото» (если камера недоступна — выбери снимок QR)

- **SFP-порты (оптика)** — при создании свитча задаёшь медные RJ45 + SFP отдельно (напр. 48 + 4). SFP рисуются отдельным блоком с синим отливом и подписью S1–S4, по смыслу это аплинки (транк).

## Обновление (v3) — к релизу
- **Локализация RU / UZ / EN** — переключатель языка в меню и на экране входа (основные экраны)
- **Онбординг** — приветствие при первом запуске + кнопка «Загрузить демо»
- **Напоминание о бэкапе** — баннер на главной, если давно не делал резервную копию (данные в localStorage)
- **Журнал изменений** — кто/что/когда менял (меню → Журнал изменений)
- **privacy.html** — политика конфиденциальности (размести рядом с приложением, дай ссылку в Play Console)
- Папка `portmap-pro-play/` — feature graphic 1024×500, иконка 512, тексты листинга RU/EN
